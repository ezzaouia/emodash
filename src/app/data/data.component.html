Data Acquisition

Emotions classification from video stream

The data is fully gathered in authentic and non-intrusive settings. Each session may have one or more archive folder(s) depending on whether or not there were any interruptions during the session, for instance, when a technical problem occurs, connection lost etc.

Recording are stored along the session in the Cloud and, once the session is terminated, the recordings are downloaded and processed as illustrated in the figure below to classify emotions from the video. Emotions are then stored in data base.

We use Microsoft emotion API from Azure Platform for Emotion classification.

  1. Pre-processing: mainly to cleanup, alignement and concatenation (if a session has many archives etc.).
  2. Video editing: as the video stream is recorded using webm (VP8 codec), stacking video streams of learner and tutor side by side result in a video desynchroniztion issues. Converting videos into .mp4 was necessary to fix the issue.
  3. Emotion classification: once the video is compiled, first frames are extracted, using a fixed sampling, of four frames per second (4FPS) and then a call to MS API is made for each frame to classify the face associate emotions. JSON results are save in data base. Parallel calls to MS API reduced processing time by almost 3 times.
